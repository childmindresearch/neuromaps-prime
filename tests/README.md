# Testing

## Running tests

```bash
# Unit tests only
pytest -m unit

# Integration tests
pytest -m integration

# Everything
pytest
```

## Test tiers

Directory-based markers are auto-applied by `conftest.py` based on the test file location.

| Marker | Directory | Description |
|--------|-----------|-------------|
| `unit` | `tests/unit/` | Pure logic tests, no external tools or data needed |
| `integration` | `tests/integration/` | Integration tests - functionality with real data |
| `regression` | `tests/regression/` | Regression tests - performance/correctness with real data |

## Directory structure

```
tests/
├── conftest.py              # Shared fixtures, markers, and CLI options
├── unit/                    # Pure logic tests, no external tools needed
│   ├── graph/               # Graph model and operation tests
│   │   ├── test_cache.py
│   │   ├── test_graph.py
│   │   ├── test_model.py
│   │   └── test_ops.py
│   ├── transformers/        # Transformer unit tests
│   │   ├── test_surf_to_surf.py
│   │   ├── test_surf_to_vol.py
│   │   ├── test_vol_to_surf.py
│   │   └── test_vol_to_vol.py
│   ├── transforms/          # Transform utility tests
│   │   ├── test_surface.py
│   │   ├── test_utils.py
│   │   └── test_volume.py
│   └── test_niwrap.py
└── integration/             # Tests requiring external tools and real data
    ├── test_graph.py        # Graph initialization, surface-to-surface, volume-to-volume
    └── transforms/
        ├── test_surface.py  # Surface sphere project/unproject via Workbench
        └── test_volume.py   # Volumetric transforms via ANTs
```

## Real data testing

By default, unit tests use temporary mock files generated by `conftest.py`. Tests
requiring real data will fetch required data, as needed, from remotely hosted locations
indicated.

## Coverage

```bash
pytest --cov=src --cov-report=term-missing
```
