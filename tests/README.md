# Testing

## Running tests

```bash
# Unit tests only (fast, no runner needed)
pytest -m unit

# Integration tests
pytest -m integration --runner docker

# Everything
pytest --runner docker
```

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--runner` | `local` | Styx runner type: `local`, `docker`, `singularity` |
| `--data-dir` | `None` | Directory where real test data is located. If not provided, temporary mock files are used. |

## Test tiers

Directory-based markers are auto-applied by `conftest.py` based on the test file location.

| Marker | Directory | Description |
|--------|-----------|-------------|
| `unit` | `tests/unit/` | Pure logic tests, no external tools or data needed |
| `integration` | `tests/integration/` | Integration tests - functionality with real data |
| `regression` | `tests/regression/` | Regression tests - performance/correctness with real data |

```
│   │   ├── test_graph.py
│   │   ├── test_model.py

│   ├── transformers/        # Transformer unit tests
│   │   ├── test_surf_to_surf.py
│   │   ├── test_surf_to_vol.py
│   │   ├── test_vol_to_surf.py
│   │   └── test_vol_to_vol.py

│   │   ├── test_surface.py

```

## Real data testing

By default, tests use temporary mock files generated by `conftest.py`. To run tests against real data, pass the `--data-dir` option:

```bash
pytest -m integration --runner docker --data-dir /path/to/test/data
```

The `--data-dir` flag expects a directory containing the data files referenced in the graph (e.g. `share/Inputs/...`).

## Coverage

```bash
pytest --cov=src --cov-report=term-missing
```
